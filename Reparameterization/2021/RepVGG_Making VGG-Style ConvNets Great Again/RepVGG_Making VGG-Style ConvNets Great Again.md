## 3. Building RepVGG via Structural Re-param

### 3.1 Simple is Fast, Memory-economical, Flexible


**Fast**

최근의 많은 멀티 브랜치 아키텍처는 VGG보다 이론적 FLOP가 낮지만 더 빠르게 실행되지 않을 수 있습니다. 예를 들어, VGG-16은 EfficientNet-B3[35]로 $8.4 \times$의 FLOP를 갖지만 1080Ti(표. 4)에서 $1.8 \times$배 더 빠르게 실행되며, 이는 전자의 계산 밀도가 후자보다 $1.5 \times$임을 의미합니다. Winograd conv에 의해 발생하는 가속을 제외하고, FLOP와 속도 간의 불일치는 속도에 상당한 영향을 미치지만 FLOP에서 고려하지 않는 두 가지 중요한 요소, 즉 메모리 액세스 비용(MAC)과 병렬 처리 정도에 기인할 수 있습니다[24]. 예를 들어, 분기 추가 또는 연결에 필요한 계산은 무시할 수 있지만 MAC은 중요합니다. 또한 MAC은 그룹별 컨볼루션에서 시간 사용량의 많은 부분을 차지합니다. 반면에, 높은 수준의 병렬 처리를 가진 모델은 동일한 FLOP에서 낮은 병렬 처리 수준을 가진 다른 모델보다 훨씬 빠를 수 있습니다. 다중 분기 토폴로지가 Inception 및 자동 생성 아키텍처에서 널리 채택됨에 따라 몇 개의 큰 연산자 대신 여러 개의 작은 연산자가 사용됩니다. 이전 연구[24]에 따르면 NASNET-A[43]의 단편화된 연산자(즉, 하나의 빌딩 블록에서 개별 conv 또는 풀링 연산의 수)의 수가 13개이며, 이는 GPU와 같은 강력한 병렬 컴퓨팅 성능을 가진 장치에 적합하지 않으며 커널 시작 및 동기화와 같은 추가 오버헤드를 유발합니다. 대조적으로, 이 숫자는 ResNets에서 2 또는 3이며 1로 만듭니다

**Memory-economical**

다중 분기 토폴로지는 모든 분기의 결과를 추가 또는 연결까지 유지해야 하기 때문에 메모리 비효율적이며, 이로 인해 메모리 점유의 최대값이 크게 증가합니다. 그림 3은 잔류 블록에 대한 입력이 덧셈까지 유지되어야 함을 보여줍니다. 블록이 기능 맵 크기를 유지한다고 가정하면 추가 메모리 점유의 피크 값은 입력으로 2입니다. 대조적으로, 일반 토폴로지를 사용하면 특정 계층에 대한 입력이 차지하는 메모리를 작업이 완료되면 즉시 해제할 수 있습니다. 특수 하드웨어를 설계할 때, 일반 ConvNet은 심층적인 메모리 최적화를 허용하고 메모리 유닛의 비용을 줄여 칩에 더 많은 컴퓨팅 유닛을 통합할 수 있습니다.

**Flexible**

다중 분기 토폴로지는 아키텍처 사양에 제약을 가합니다. 예를 들어, ResNet은 conv 레이어를 resid ual 블록으로 구성해야 하는데, 이는 모든 잔차 블록의 마지막 conv 레이어가 동일한 모양의 텐서를 생성해야 하거나 바로 가기 덧셈이 의미가 없기 때문에 유연성을 제한합니다. 설상가상으로, 다중 분기 토폴로지는 일부 중요하지 않은 채널을 제거하는 실용적인 기술인 채널 가지치기의 적용을 제한하고[22, 14], 일부 방법은 각 레이어의 적절한 너비를 자동으로 검색하여 모델 구조를 최적화할 수 있습니다[8]. 그러나 다중 분기 모델은 가지치기를 까다롭게 만들고 성능이 크게 저하되거나 가속 비율이 낮아집니다[7, 22, 9]. 대조적으로, 일반 아키텍처를 사용하면 요구 사항에 따라 모든 conv 계층을 자유롭게 구성하고 더 나은 성능-효율성 균형을 얻기 위해 정리할 수 있습니다.

### 3.2 Training-time Multi-branch Architecture

플레인 ConvNet은 많은 강점을 가지고 있지만 한 가지 치명적인 약점이 있습니다. 예를 들어, BN[19]과 같은 mod ern 구성 요소를 사용하면 VGG-16은 ImageNet에서 72% 이상의 top-1 정확도에 도달할 수 있습니다. 우리의 구조적 재매개변수화 방법은 정보 흐름을 y = x + f(x)로 모델링하기 위해 바로 가기 분기를 명시적으로 구성하고 잔차 블록을 사용하여 f를 학습하는 ResNet에서 영감을 받았습니다. x와 f(x)의 차원이 일치하지 않으면 y = g(x)+f(x)가 되며, 여기서 g(x)는 1 1 conv에 의해 구현되는 합성곱 바로 가기입니다. ResNets의 성공에 대한 설명은 이러한 다중 분기 아키텍처가 모델을 수많은 얕은 모델의 암시적 앙상블로 만든다는 것입니다[36]. 구체적으로 말하자면, n 블록이 있는 경우, 모든 블록이 흐름을 두 개의 경로로 분기하기 때문에 모델은 2n 모델의 앙상블로 해석될 수 있습니다. 다중 분기 토폴로지에는 단점이 있지만 분기가 학습에 도움이 되는 것처럼 보이기 때문에[36], 여러 분기를 사용하여 수많은 모델의 학습 시간만 만듭니다. 대부분의 메모리를 더 얕거나 간단하게 만들기 위해 ResNet과 같은 ID(차원이 일치하는 경우에만)와 1 1 분기를 사용하여 빌딩 블록의 학습 시간 정보 흐름이 y = x +g(x) +f(x)가 되도록 합니다. 우리는 단순히 훈련 시간 모델을 구성하기 위해 이러한 블록을 여러 개 쌓습니다. [36]과 동일한 관점에서 모델은 n개의 블록을 가진 3n개의 멤버로 구성된 앙상블이 됩니다.
